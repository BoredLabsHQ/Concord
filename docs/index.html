Product Overview

Concord is an AI-powered plugin designed for messaging platforms like Matrix, Telegram, and Discord. It enhances public channel discovery based on content relevance and serendipity, connecting communities discussing similar ideas across different platforms. Key features include:

  • AI-Powered Topic Mapping: Helps community organizers identify trending discussions and coordinate efforts.
  • Conversation Assistance: The AI follows conversations, suggesting next steps and assisting with scheduling.
  • Decentralized Identity and Access Control: Implements decentralized identity standards and smart contracts for secure authentication and governance.
  • Blockchain-Based Reputation System: Introduces an on-chain reputation mechanism to build trust within the community.
  • Open-Source and Decentralized Storage: Utilizes decentralized storage solutions for data resilience and censorship resistance.

Key Features and Considerations

1. Graph Channel Structure for Public Channels

  • Unique Identification:
    • Hierarchical Paths: Utilize a hierarchical naming system (e.g., /root/branch/leaf) to uniquely identify channels.
    • ENS Integration: Use the Ethereum Name Service to provide human-readable names for channels, enhancing discoverability and ease of use.
    • Channel IDs: Assign unique identifiers to channels for easy reference and interaction.
  • Channel Management:
    • Creation Rights: Allow any user to create channels but implement moderation mechanisms via smart contracts to prevent spam or abuse.
    • Moderation Policies: Define clear policies enforced by smart contracts to maintain community standards.

2. Access Control and Governance

  • Smart Contracts for Access Control:
    • Decentralized Governance: Deploy smart contracts on Ethereum to manage permissions for channel creation, access rights, and moderation.
    • Transparent Policies: All rules and policies are transparent and enforceable on the blockchain, ensuring fairness and trust.
  • Moderation Mechanisms:
    • Immutable Rules: Automate moderation policies through smart contracts to prevent abuse and spam.
    • Community Participation: Establish a Decentralized Autonomous Organization (DAO) to allow community-driven governance and decision-making.
  • Role-Based Access:
    • Verifiable Credentials: Issue credentials that attest to user roles, permissions, or attributes within communities.
    • Access Levels: Define different access levels (e.g., admin, moderator, member) managed through smart contracts.

3. Decentralized Identity and Authentication

  • User Authentication:
    • Ethereum Wallets: Utilize wallets like MetaMask for users connecting via Ethereum accounts.
    • DID Standards: Implement W3C's Decentralized Identifiers (DIDs) for user identity management using the did:ethr method.
    • Verifiable Credentials: Leverage credentials to verify user identity and attributes without exposing personal data.
  • Alternative Methods:
    • Phone or Email Onboarding: Allow onboarding with phone or email for users who prefer traditional methods.
    • Bridging Identities: Enable users to link traditional accounts with decentralized identities for a seamless experience.
  • Privacy and Control:
    • User-Controlled Data: Users maintain control over their identity data without relying on centralized authorities.
    • Selective Disclosure: Allow users to share only necessary information, enhancing privacy.

4. Message Encryption and Decentralized Storage

  • Message Encryption:
    • Public Key Cryptography: Use users' public keys to encrypt messages, ensuring secure communication.
    • End-to-End Encryption (E2EE): Ensure only intended recipients can decrypt messages, protecting against eavesdropping.
  • Decentralized Storage Solutions:
    • IPFS or Swarm: Utilize decentralized storage for messages and files, enhancing data resilience.
    • Content Addressing: Store data using content-addressable storage for integrity and permanence.
    • Censorship Resistance: Eliminates reliance on centralized servers, making data tampering difficult.
  • Key Management Based on User Onboarding:
    • Ethereum Users:
      • Key Usage: When connecting with Ethereum, the user's existing Ethereum account keys are used for authentication and encryption.
    • Phone or Email Onboarding:
      • User-Generated Keys: Users can set up their own encryption keys during onboarding.
    • Custodial Key Management:
      • Backend-Managed Keys: For users who prefer a custodial system, the backend will generate and manage encryption keys on their behalf.
      • Ease of Use: Simplifies the process for users uncomfortable with key management.
  • Security Measures:
    • Private Key Protection: Ensure that private keys are securely stored and never leave the user's device unless under the custodial system.
    • Key Recovery Solutions: Provide options for key recovery in case users lose access, such as mnemonic phrases or hardware tokens.

5. AI-Powered Semantic Analysis

  • Data Processing:
    • Public Channels Only: Respect privacy by limiting AI processing to public data.
    • Anonymization: Ensure user data is anonymized if necessary to protect identities.
  • Semantic Value Calculation:
    • Natural Language Processing (NLP): Employ NLP techniques to understand context and topics.
    • Machine Learning Models: Use models like BERTopic and Llama3.1 to analyze content.
    • Topic Mapping: Visualize relationships between topics and channels to aid discovery.
  • Recommendations and Discovery:
    • Similar Channels: Suggest channels with related discussions to users based on their interests.
    • Content Highlights: Summarize popular or significant discussions to keep users informed.
    • Serendipitous Discovery: Introduce users to new topics and communities they might find interesting.

6. Blockchain-Based Reputation System

  • On-Chain Reputation Scores:
    • Reputation Metrics: Implement reputation metrics that are stored and calculated on the Ethereum blockchain.
    • Transparency: Reputation metrics are tamper-proof and publicly verifiable, enhancing trust.
  • Incentivized Moderation:
    • Positive Contributions: Reward users who contribute positively to the community.
    • Spam Prevention: Deter malicious activities by associating actions with reputation scores.
  • Trust Building:
    • User Trust: Helps users identify trustworthy community members.
    • Community Engagement: Encourages active participation and fosters a vibrant community.
  • Reputation-Based Access:
    • Exclusive Features: Unlock advanced features or privileges based on reputation.
    • Moderation Roles: High-reputation users can be granted moderation capabilities.
